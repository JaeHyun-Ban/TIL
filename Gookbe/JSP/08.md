# 예외 페이지

---

## 1. 예외 종류

예외의 상황이 발생할 때 웹컨테이너(톰캣)에서 제공하는 기본적으로 보여주는 페이지이다

200번대 예외 - 성공(예외가 아니잖아..?)
400번대 예외 - 브라우저의 예외
500번대 예외 - 서버측 예외

---

## 2. 처리방법

1. 직접 예외 처리
   try~catch로 직접 처리

   > 너무 노가다

2. 에러를 처리할 페이지를 따로 지정
   지시자의 `errorPage`속성을 사용해 지정한다
   `<%@ page isErrorPage="true" %>`

3. 응답 상태 코드별로 에러 페이지 지정하기
   web.xml파일에 error-page처리를 작성 할 수 있다.

에러 페이지 우선순위
error
